# ROS2 States 관찰 ### 주의 * conda jazzy 환경에서 코드를 진행할것 * flask를 사용할 것 * ros2 jazzy 버전을 사용할 것 * front-end는 밝은 톤일 것 * 아래 내용의 실행과 테스트를 위해 ros2의 demo_node_cpp의 talker나 turtlesim을 적절히 활용해줘 ### 내용 * 웹브라우저에서 현재 ROS2의 상태를 확인 * 먼저, ROS_DOMAIN_ID를 지정하고, 활성화 버튼을 누르도록 * 한 세션에서는 topic의 리스트를 보여줘. 이때 Hz와 데이터 타입도 보여줘. topic을 구독하거나, 발행하는게 아니야. 상황만 관찰하는 거야. 그리고, topic이 pub인지 sub인지도 확인해줘. * 또 한 세션에서는 service를 보여줘. 역시 데이터 타입도 보여줘 * 역시 action도 데이터 타입과 함께 보여줘 * 그리고 또 한 세션에는 tf도 받아서 구독하도록 해줘. 단, tf는 개괄적이라도 parent와 child의 관계가 보이도록 해줘 * 화면을 간결하게 구성해줘.

Code ros_monitor.py 작성
ROS2 모니터링 핵심 모듈 구현:
  • RosMonitor 클래스 — activate/deactivate로 rclpy 노드 생명주기 관리
  • Graph API — get_topic_names_and_types(), get_publishers_info_by_topic() 등으로 구독 없이 토픽/서비스 관찰
  • Hz 측정 — ros2 topic hz subprocess를 백그라운드에서 순차 실행
  • TF 구독 — /tf, /tf_static 구독으로 parent-child 트리 구축
  • 내부 노드 필터링 — ros_web_monitor, _ros2cli_* 노드를 pub/sub 카운트에서 제외

Code app.py Flask 백엔드 작성
API 엔드포인트 구현:
  • POST /api/activate — domain ID 설정 후 모니터링 시작
  • POST /api/deactivate — 모니터링 중지
  • GET /api/topics — 토픽 목록 (타입, pub/sub, Hz)
  • GET /api/services — 서비스 목록 (타입)
  • GET /api/actions — 액션 목록 (타입)
  • GET /api/tf — TF 트리 (parent-child)

Fix 포트 충돌 해결
macOS AirPlay Receiver가 포트 5000을 점유하고 있어 서버 시작 실패. 포트를 5050으로 변경하여 해결.
Fix Hz 측정 로직 수정
ros2 topic hz는 종료되지 않는 프로세스이므로 subprocess.run(timeout=5)가 예외를 발생시키며 출력을 버림.
subprocess.Popen + communicate(timeout=4)로 변경하여, 타임아웃 시 프로세스를 kill하고 그때까지의 출력에서 average rate: X.XXX를 파싱하도록 수정.
Fix Graph 오염 문제 해결
Hz 측정용 ros2 topic hz subprocess들이 임시 노드(_ros2cli_*)를 생성하여 pub/sub 카운트와 서비스 목록을 오염.
_is_internal() 필터 함수를 도입하여 ros_web_monitor 및 _ros2cli_* 노드를 모든 카운트와 목록에서 제외.
Fix Action 감지 방식 변경
ros2 action list -t 명령이 무한 대기(hang) 현상 발생.
대안으로 rclpy graph API의 서비스 목록에서 /_action/send_goal 패턴을 찾아 액션을 감지하도록 변경. 타입은 _SendGoal 접미사를 제거하여 추출.
Action 내부 토픽(/_action/feedback, /_action/status)과 서비스도 일반 목록에서 필터링.
Test 통합 테스트
